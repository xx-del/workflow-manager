# Expander 一对多映射模式

## 问题

_index.yaml 定义7个节点，WORKFLOW.md 有12个步骤。节点与步骤不是一对一关系：

```
_index.yaml节点          WORKFLOW.md步骤         步骤数
──────────────────────────────────────────────────────
step_1 (启动扫描)     →  步骤0-5              →  6步骤
step_2 (断点返回)     →  步骤5.5             →  1步骤
step_3 (WIH下载)      →  步骤6               →  1步骤
step_4 (AWVS下载)     →  步骤7               →  1步骤
step_5 (JS分析)       →  步骤10              →  1步骤
step_6 (AWVS分析)     →  步骤11              →  1步骤
step_7 (清理任务)     →  步骤8-9             →  2步骤
```

## 匹配规则

5条规则，按优先级：

1. **task字段匹配**（最高优先级）：步骤名称包含节点task → 归属该节点
2. **步骤关键词匹配**：去掉编号后的步骤关键词在节点名中
3. **节点关键词匹配**：去掉括号的节点关键词在步骤名中（≥3字符）
4. **断点特殊匹配**：type=breakpoint节点，匹配"启动心跳"/"核心断点"/"断点返回"
5. **占位步骤**：断点节点无匹配步骤时，自动创建占位步骤

## 关键陷阱

- 断点节点的匹配规则不能太宽泛（如"心跳"会匹配所有心跳步骤）
- 排除断点节点参与通用匹配，避免误归属
- 未匹配步骤标记为type=action（默认类型）
