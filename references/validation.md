# 工作流校验规范

## 概述

工作流校验是深度模拟测试，由 AI 扮演虚拟执行器，分析工作流中每个命令、代码、文档，判断逻辑上是否能完整执行。

## 触发方式

| 触发词 | 说明 |
|--------|------|
| "校验工作流" | 完整校验 |
| "验证工作流" | 完整校验 |
| "检查工作流" | 完整校验 |
| "测试工作流" | 基础测试（实际执行前） |

## 校验流程

```
【Phase 1: 基础校验】（工具自动）
├── 结构完整性
│   - WORKFLOW.md 格式正确
│   - 必需字段存在
│   - YAML 语法正确
│
├── 依赖分析
│   - 步骤依赖形成 DAG（无环）
│   - 参数引用有效
│   - 输出文件被使用
│
└── 配置完整性
    - retry 配置存在
    - notify 配置存在
    - guardian 配置存在

【Phase 2: 深度模拟】（AI 分析）
├── 命令分析
│   - 命令语法正确性
│   - 参数逻辑合理性
│   - 环境依赖可行性
│   - 错误处理完备性
│
├── 代码分析
│   - 代码逻辑正确性
│   - 输入输出匹配
│   - 异常处理完备
│   - 依赖库可用
│
├── 文档分析
│   - 引用路径存在
│   - 文档格式正确
│   - 内容与步骤匹配
│
├── 参数分析
│   - 参数传递正确
│   - 默认值合理
│   - 类型匹配
│
└── 流程分析
    - 执行路径完整
    - 分支覆盖合理
    - 终止条件明确

【Phase 3: 诊断报告】
├── 问题汇总
│   - 🔴 错误：无法执行
│   - 🟡 警告：可能失败
│   - 🔵 建议：可优化
│
└── 修复方案
    - 问题根因分析
    - 修复建议
    - 修改后预览
```

## AI 分析方法

### 命令分析

```markdown
## 分析命令: curl http://api/data?date={{date_range}} > result.json

### 语法检查
- curl 语法正确
- URL 格式正确
- 重定向语法正确

### 逻辑分析
- {{date_range}} 参数需要替换
- 输出文件 result.json
- 需要 API 端点可访问

### 环境依赖
- curl 已安装
- 网络可达
- 有写入权限

### 潜在问题
- ⚠️ {{date_range}} 默认值未定义
- ⚠️ 未指定超时时间
- ⚠️ 无错误处理（网络失败时）

### 建议
- 添加 --connect-timeout 参数
- 添加错误处理：|| exit 1
```

### 代码分析

```markdown
## 分析代码: python process.py --input result.json --output output.txt

### 代码逻辑
- 读取 result.json
- 处理数据
- 输出到 output.txt

### 输入输出
- 输入: result.json（来自步骤 1）
- 输出: output.txt

### 依赖分析
- Python 3.x
- process.py 存在
- 依赖库（pandas, json...）

### 潜在问题
- 🔴 result.json 可能不存在（步骤 1 失败时）
- 🟡 未指定 Python 版本
- 🔵 可添加 --verbose 参数调试

### 建议
- 添加输入文件存在性检查
- 指定 Python 版本: python3
```

### 流程分析

```markdown
## 分析流程

### 执行路径
步骤 1: 获取数据
  ↓ (result.json)
步骤 2: 处理数据
  ↓ (output.txt)
完成

### 依赖链验证
- 步骤 1 → 步骤 2: ✅ result.json
- 步骤 2 无后续依赖

### 分支分析
- 无分支，线性流程
- 所有步骤必执行

### 终止条件
- 步骤 2 完成即终止
- ✅ 明确
```

## 校验规则库

### 命令规则

| 命令 | 检查项 |
|------|--------|
| curl | URL 格式、超时、错误处理 |
| python | 文件存在、依赖库、版本 |
| ssh | 主机可达、密钥权限 |
| git | 仓库存在、权限 |

### 工具规则

| 工具 | 检查项 |
|------|--------|
| terminal | 命令语法、环境依赖 |
| read_file | 文件路径、权限 |
| write_file | 目录存在、权限 |
| browser_* | URL 有效、选择器存在 |

### 通用规则

| 规则 | 说明 |
|------|------|
| 参数替换 | {{param}} 必须有定义或默认值 |
| 文件依赖 | 上一步输出必须匹配下一步输入 |
| 错误处理 | 网络操作必须有容错机制 |
| 权限检查 | 涉及写入需要权限检查 |
