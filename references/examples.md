## 执行工作流

### 用户
"执行漏扫"

### AI
> ⚠️ **必须先思考分析，然后指挥 agent-pool 执行，禁止直接执行命令**

```
[*] 思考分析阶段（v3.5新增）
    ├─ 任务类型分析:
    │   ├─ CLI任务: 步骤1、2、4、5（确定性高）
    │   └─ 网络任务: 步骤3（耗时长）
    ├─ 依赖关系分析:
    │   ├─ 步骤1 → 步骤2（需要最新POC）
    │   ├─ 步骤2 → 步骤3（需要URL列表）
    │   └─ 步骤3 → 步骤4（需要扫描结果）
    │   → 结论: 必须串行执行
    ├─ Agent分配策略:
    │   ├─ 方案A: 1个agent执行所有步骤（简单但慢）
    │   ├─ 方案B: 3个agent各负责一步（并行但依赖复杂）
    │   └─ 方案C: 2个agent（CLI agent + 扫描agent）
    │   → 选择方案C（职责清晰，复用性高）
    └─ 风险预判:
        ├─ 步骤3可能超时（扫描耗时长）
        └─ 网络错误自动重试

[✓] 分配方案:
    ├─ agent-1 (CLI执行): 步骤1、2、4、5
    └─ agent-2 (扫描执行): 步骤3

[→] 指挥 agent-pool 执行...
    ├─ 传递任务列表
    ├─ 传递分配策略
    ├─ 传递守护语义
    └─ 监控状态...
```
