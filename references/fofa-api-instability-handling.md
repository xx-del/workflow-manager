# FOFA API 不稳定问题诊断与处理

## 问题描述

执行大批量关键词搜索时（100+ 关键词），FOFA API 频繁返回错误。

## 错误类型

### 1. 服务端错误（-501）

```
FOFA API错误: [-501] 服务错误,请稍候重试
```

**原因**：FOFA 服务端问题，非本地配置问题。

### 2. 请求超时

```
请求超时: https://fofa.info/api/v1/search/all
```

**重试机制**：
- 每次查询超时：30 秒
- 超时后重试：3 次
- 每次重试等待：30 秒
- 单个失败关键词总耗时：约 90 秒

## 影响范围

**案例**（2026-05-18）：
- 关键词总数：146 个
- 失败率：约 50%
- 从批次 35 开始（约第 100 个关键词）
- 失败关键词示例：
  - 电缆、架空线路、杆塔、绝缘子
  - 继电保护、测控装置、保护定值
  - 差动保护、距离保护、零序保护
  - PMU、故障录波、后台监控

## 诊断方法

### 1. 检查后台进程日志

```python
process(action='log', session_id='proc_xxx', limit=100)
```

**关键词**：
- `FOFA API错误: [-501]`
- `请求超时，重试`
- `查询失败`

### 2. 统计失败率

```bash
# 统计 -501 错误数量
grep "FOFA API错误: \[-501\]" log.txt | wc -l

# 统计超时数量
grep "请求超时" log.txt | wc -l
```

## 解决方案

### 方案 A：使用后台进程（推荐）

```python
terminal(
    command="cd /path && python3 main.py",
    background=True,
    notify_on_complete=True
)
```

**优势**：
- 无超时限制
- 进程持续运行
- 完成后自动通知

### 方案 B：分批执行

```
第1批：1-50 个关键词（约 250 秒）
第2批：51-100 个关键词（约 250 秒）
第3批：101-146 个关键词（约 230 秒）
```

### 方案 C：筛选高价值关键词

根据历史分析，只搜索高价值关键词：
- 储能、风电、光伏发电、逆变器
- EMS、SCADA、调度中心
- 继电保护、PMU
- 智能电网、电力调度

## 预防措施

1. **监控进程状态**：定期检查后台进程日志
2. **预期失败率**：FOFA API 不稳定时，预计 30-50% 失败率
3. **结果验证**：完成后检查 JSON 报告大小，判断是否正常

## 时间估算

**正常情况**：
- 单个关键词查询：约 5 秒
- 146 个关键词：约 730 秒（12 分钟）

**API 不稳定时**：
- 失败关键词：约 90 秒（超时+重试）
- 50% 失败率：约 45 分钟

## 结果偏少原因

1. **FOFA API 不稳定**：约 50% 关键词查询失败
2. **去重**：大部分结果已存在于历史数据库
3. **时间范围限制**：一个月限制较严格
4. **title 字段搜索**：部分专业术语不在 title 中出现
