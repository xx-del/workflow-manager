# 技能文档优化实践

**日期**: 2026-05-14
**触发**: SKILL.md 引用过时代码（Python hook 文件）

---

## 问题诊断

| 问题 | 表现 | 根因 |
|------|------|------|
| 引用过时代码 | SKILL.md 引用 Python hook，实际使用 shell | 未及时清理遗留代码 |
| 文档定位错误 | 代码解读 + 版本历史手册 | 未明确"使用说明"定位 |
| 篇幅过长 | ~10,000 字符，70+ references | 累积历史未清理 |

---

## 优化原则

**定位**：AI 使用说明书（非版本历史手册）

**使用者视角**：告诉"如何做"（非"为什么"）

**篇幅控制**：≤ 3000 字符

**删除项**：
- 版本号、架构陷阱、设计记录
- 代码解读、返回值协议、最佳实践
- references 文档引用（代码已实现）

**保留项**：
- 初始化/执行/完成命令
- AI 职责清单
- 核心约束（禁止事项）
- 工作流规范模板

---

## 执行步骤

1. 备份当前 SKILL.md
2. 删除遗留代码（未使用的 Python hook 文件）
3. 写入优化后内容
4. 清理缓存和备份文件

---

## 验证标准

| 指标 | 标准 |
|------|------|
| 字符数 | ≤ 3000 |
| 章节数 | ≤ 5 |
| references 引用 | 0（代码已实现） |
| 版本历史 | 无 |
| 代码解读 | 无 |

---

## 教训

**根因**：文档随时间累积，未定期清理

**预防**：
- 定期审计技能文档
- 发现引用过时代码立即清理
- 保持"使用说明"定位

**关键判断**：
- 如果内容是"为什么这样设计" → 删除
- 如果内容是"代码如何实现" → 删除
- 如果内容是"AI 如何操作" → 保留
