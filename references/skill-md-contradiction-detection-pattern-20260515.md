# SKILL.md 内部矛盾检测模式

**日期**: 2026-05-15
**发现来源**: 拼接工作流执行方案分析

---

## 问题现象

Hook 注入了正确的指令，但主AI没有执行，而是按照 SKILL.md 中另一个章节的描述执行。

---

## 根因分析

SKILL.md 中存在**两套互相矛盾的执行方案**：

| 章节 | 位置 | 内容 | 方向 |
|------|------|------|------|
| 第四章 | 第78-90行 | "逐个执行子工作流" | 分散执行 |
| 第十章 | 第336-340行 | "合并生成统一status.md" | 合并执行 |

**因果链**：
```
主AI 先读第四章 → 按分散执行理解
    ↓
Hook 注入第十章方案（合并执行）
    ↓
主AI已被第四章引导 → 忽略/覆盖 Hook 注入
    ↓
结果：实际执行与 Hook 注入不一致
```

---

## 检测方法

### 1. 搜索关键词冲突

```bash
# 检测同一概念的不同描述
grep -n "逐个执行" SKILL.md
grep -n "合并" SKILL.md
```

### 2. 章节一致性检查

对于同一主题（如"拼接工作流"），检查所有相关章节：
- 是否描述一致？
- 是否有新旧方案混用？
- Hook 注入内容与 SKILL.md 是否匹配？

### 3. 执行结果验证

实际执行结果与 Hook 注入内容对比：
- Hook 说"合并" → 实际是否合并？
- Hook 说"逐个" → 实际是否逐个？

---

## 修复原则

1. **确定正确方案**：Hook 注入通常是最新设计，优先遵循
2. **统一所有章节**：删除或更新旧方案的描述
3. **同步相关文档**：references/ 下的执行实例文档也要同步
