# 死代码检测方法论

## 概述

本文档记录了 workflow-manager 技能的代码质量分析方法论，包括如何检测死代码、分析根因、制定优化方案。

---

## 检测方法论

### 1. 代码-文档一致性分析

**步骤**：
1. 列出所有代码文件和函数定义
2. 检查每个定义是否被调用
3. 对比文档描述与实际实现
4. 识别不一致项

**关键检查点**：
- 模块是否被 import
- 类是否被实例化
- 方法是否被调用
- 返回值是否被使用

### 2. 死代码分类

| 类型 | 特征 | 示例 |
|------|------|------|
| **架构变更遗留** | 旧架构被替代，代码未删除 | guardian 模块 |
| **集成不完整** | 代码已实现，未集成到接口 | extractor/validator/optimizer |
| **调用链断裂** | 实例化但方法未调用 | consolidator |
| **方法未调用** | 定义但从未使用 | _auto_optimize() |

### 3. 根因分析框架

**四类根因**：

1. **架构变更遗留**：旧代码未清理
2. **集成不完整**：实现完成但未接入
3. **调用链断裂**：中间环节缺失
4. **方法未调用**：规划未实现

### 4. 优化方案制定

**优先级定义**：
- P0（立即处理）：架构遗留、死代码
- P1（短期处理）：功能激活、调用链修复

**方案类型**：
- 删除：架构遗留代码
- 集成：未接入功能
- 修复：断裂调用链
- 保留：文档标注废弃

## 实际案例

### 案例：workflow-manager 死代码清理

**发现**：52.2% 代码（5664 行）未使用

**根因分析**：
```
| 模块 | 行数 | 根因 | 解决方案 |
|------|------|------|----------|
| guardian | 1115 | 架构变更遗留 | 删除 |
| extractor | 3209 | 集成不完整 | 添加文档触发 |
| validator | 713 | 集成不完整 | 添加文档触发 |
| optimizer | 627 | 集成不完整 | 添加文档触发 |
| consolidator | ~180 | 调用链断裂 | 集成到 finalize |
```

**优化结果**：
- 代码减少：1353 行（-12.4%）
- 死代码清除：100%
- 功能激活：4 个核心功能

## 检测命令速查

```bash
# 检查模块是否被引用
grep -r "from module\|import module" --include="*.py" | grep -v "^./module/"

# 检查类是否被实例化
grep -r "ClassName()" --include="*.py" | grep -v "class ClassName"

# 检查方法是否被调用
grep -r "method_name" file.py | grep -v "def method_name"

# 统计代码行数
find . -name "*.py" -exec wc -l {} + | tail -1
```

## 教训

1. **架构变更必须清理旧代码**：避免维护负担
2. **功能实现必须完整集成**：避免"实现-集成断层"
3. **定期代码审计**：使用本方法论检测死代码
4. **文档与代码同步**：确保一致性
