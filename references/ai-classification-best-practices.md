# AI 智能分类大量 URL 的最佳实践

## 问题：子 agent 工具限制

**症状**：子 agent 使用 `read_file` 读取大 JSON 文件（30-40KB 单行）时被截断，无法获取完整数据。

**根因**：`read_file` 工具对大文件有截断限制。

**解决方案**：
1. **主 agent 预处理**：用 `jq` 提取关键字段到文本文件
   ```bash
   jq -r '.[] | "\(.url)|\(.title)|\(.footer_text)|\(.domain)"' all_assets.json > url_list.txt
   ```
2. **分割批次**：用 `split` 命令分割为小批次
   ```bash
   split -l 26 url_list.txt batch_url_
   ```
3. **子 agent 分析**：每个子 agent 处理一个小批次（~26 行）

## 三问判断法（资产分类）

**核心逻辑**：以国家电网为中心 → 扩散电力行业 → 排除明确不可能搭边的

| 问题 | 判断条件 | 结果 |
|------|----------|------|
| Q1 | domain 含 sgcc/stategrid，title/footer 含"国家电网"/"国网"/"95598" | 国家电网资产 |
| Q2 | 南方电网/供热/铁路/地铁/交通/燃气/水务 | 排除 |
| Q3 | 含电力关键词（变电站/配电/输电/调度/电力/电网） | 电力行业资产 |

**判断原则**：
- 关系模糊的、不好判断的 → 不排除，归入电力行业
- AI 自主判断最高，规则仅作参考

## 并行分析模式

**数据量阈值**：>30 个 URL 需要并行分析

**批次大小**：每批 26-30 个 URL

**最大并发**：3 个 delegate_task 同时运行

**执行流程**：
```
主 agent:
  1. jq 提取关键字段 → url_list.txt
  2. split 分割为 N 个批次
  3. 并行启动 min(N, 3) 个 delegate_task
  4. 汇总所有子 agent 结果
  5. 写入输出文件
```

---

## 输出文件格式（纯 URL 列表）

**⚠️ 重要**：输出文件必须是纯 URL 列表，不带任何前缀/后缀/注释

**正确示例**：
```
http://example.com
http://another.example.com
```

**错误示例**：
```
标题|http://example.com|分数
# 这是注释
国家电网资产: http://example.com
```

**原因**：后续脚本直接读取 URL 列表，无法解析带标题/分数/注释的格式。
