# 技能文档优化方法论

**创建日期**：2026-05-13  
**来源**：SKILL.md优化实践

---

## 一、核心原则

### 1.1 文档定位

**正确理解**：
- **使用说明书**：告诉用户如何使用
- **不是代码文档**：不解释代码实现细节
- **用户视角**：用户需要知道"怎么做"，不需要知道"代码怎么实现"

**错误示例**：
- ❌ 混合使用说明和代码文档
- ❌ 详细列举代码实现细节
- ❌ 架构设计说明（用户不关心）

### 1.2 精简原则

**用户偏好**（2026-05-13 明确）：
- 偏好大幅精简而非渐进优化
- 对冗余内容零容忍
- 重视文档定位清晰

**精简目标**：
- 行数精简：目标50-90%
- 内容精简：删除代码细节、架构说明、重复内容
- 结构精简：章节清晰，无冗余

## 二、优化流程

### 2.1 分析阶段

**Step 1：识别问题**
- 文档定位是否清晰？
- 是否混合代码文档和使用说明？
- 是否有重复内容？
- 是否有冗余章节？

**Step 2：分类内容**
- **保留**：使用方法、核心概念、验证清单
- **删除**：代码细节、架构说明、重复内容
- **精简**：约束说明（已注入Hook）、详细列举
- **整合**：重复章节合并

### 2.2 执行阶段

**Step 3：删除章节**
- 删除代码说明章节
- 删除架构限制章节
- 删除实现细节章节
- 删除设计约束章节

**Step 4：精简章节**
- 快速开始：详细命令 → 简洁步骤
- 职责说明：详细列举 → 核心要点
- 约束说明：详细列举 → 核心约束（已注入Hook）
- 验证机制：保留验证清单

**Step 5：整合章节**
- 整合重复内容
- 合并相似章节
- 指向详细文档

### 2.3 验证阶段

**Step 6：验证结果**
- 检查章节结构是否清晰
- 检查是否有遗漏重要内容
- 检查是否还有重复内容
- 检查行数是否达到预期

## 四、Hook迁移方法论

### 4.1 迁移原因

**问题**：
- SKILL.md中的hooks配置无执行路径
- hooks/目录下的脚本未被Hermes执行
- 实际生效的是Gateway Hooks

**解决方案**：
- 迁移到Gateway Hooks（官方支持）
- 统一配置管理
- 所有Hook都生效

### 4.2 迁移流程

**阶段1：备份**
```bash
# 备份工作流技能目录
cp -r ~/.hermes/skills/openclaw-imports/workflow-manager/ /x/AI/backup/
```

**阶段2：迁移**
```bash
# 创建Gateway Hook配置
mkdir -p ~/.hermes/hooks/<hook-name>/
cat > ~/.hermes/hooks/<hook-name>/HOOK.yaml << 'EOF'
name: <hook-name>
description: "<hook描述>"
events:
  - <事件名>
command: "/home/kali/.hermes/agent-hooks/<script-name>.sh"
EOF

# 移动脚本
mv hooks/<script>.sh /home/kali/.hermes/agent-hooks/<new-name>.sh
chmod +x /home/kali/.hermes/agent-hooks/<new-name>.sh
```

**阶段3：清理**
```bash
# 删除SKILL.md中的hooks配置
# 删除hooks/目录（已备份）
```

**阶段4：验证**
```bash
# 检查Gateway Hooks配置
ls ~/.hermes/hooks/<hook-name>/HOOK.yaml

# 检查脚本
ls ~/.hermes/agent-hooks/<script>.sh

# 测试触发
# 发送消息、调用工具、结束会话
```

### 4.3 迁移结果

**已生效的Hook**：
1. workflow-status-check（pre_llm_call）
2. workflow-step-check（pre_tool_call）
3. workflow-progress-update（post_tool_call）
4. workflow-session-cleanup（on_session_end）

**配置位置**：
- Gateway Hooks：`~/.hermes/hooks/workflow-*/`
- Hook脚本：`~/.hermes/agent-hooks/workflow-*.sh`
- 使用指南：`HOOKS_GUIDE.md`

## 五、最佳实践

### 5.1 文档编写

**原则**：
- 用户视角优先
- 使用导向明确
- 无代码实现细节
- 无架构设计说明

**检查清单**：
- [ ] 定位是否清晰？
- [ ] 是否有代码细节？
- [ ] 是否有重复内容？
- [ ] 是否易于阅读？

### 5.2 精简策略

**删除优先级**：
1. 代码实现细节（最高优先级）
2. 架构设计说明
3. 重复内容
4. 冗余章节

**精简技巧**：
- 用表格代替长段落
- 用要点代替详细说明
- 指向详细文档而非重复内容
- 保留核心，删除冗余

### 5.3 维护建议

**定期审查**：
- 频率：每月一次
- 内容：是否有新的冗余？
- 行动：及时精简

**保持精简**：
- 不添加代码细节
- 不添加架构说明
- 详细内容独立文档
- 核心内容保留

## 六、总结

### 6.1 核心价值

1. **提升可读性**：精简90%，更易阅读
2. **明确导向**：从代码文档转变为使用说明书
3. **消除重复**：整合重复内容，结构清晰
4. **保留核心**：所有使用相关内容完整保留

### 6.2 用户价值

- **快速上手**：用户快速理解如何使用
- **易于查找**：关键信息快速定位
- **无干扰**：不被代码细节干扰

### 6.3 维护价值

- **易于维护**：无重复内容
- **清晰定位**：纯使用说明书
- **独立文档**：详细内容独立维护
