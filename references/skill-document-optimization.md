# 技能文档优化方法论

**日期**: 2026-05-09
**适用范围**: 所有 SKILL.md 文档优化
**优化目标**: 清晰、简洁、可维护

---

## 优化流程

### Phase 1: 问题诊断

**步骤**：

1. **统计基本信息**
   - 文档行数
   - 文件大小
   - 章节数量
   - 关键词出现频率

2. **搜索重复内容**
   ```bash
   # 搜索关键词出现次数
   grep -c "关键词" SKILL.md
   
   # 搜索章节重复
   grep "^## " SKILL.md | sort | uniq -d
   ```

3. **分析章节结构**
   ```bash
   # 列出所有章节
   grep "^##" SKILL.md
   ```

4. **识别重复章节**
   - 同一内容出现在多个章节
   - 同一约束被重复说明
   - 验证清单重复出现

---

### Phase 2: 制定优化方案

**模板**：

```markdown
# 行为计划：[技能名称] SKILL.md 优化

## 目标
[量化目标：从 X 行优化到 Y 行，减少 Z%]

## 删除清单
| 章节 | 位置 | 原因 |
|------|------|------|
| | | 与 X 章节重复 |

## 保留清单
| 章节 | 说明 |
|------|------|
| | 核心内容 |

## 预期效果
- 文档行数：X → Y 行
- 章节数：A → B 个
- 重复内容减少：Z%
```

---

### Phase 3: 执行优化

**步骤**：

1. **备份原文档**
   ```bash
   cp SKILL.md SKILL.md.bak_$(date +%Y%m%d)
   ```

2. **删除重复章节**
   - 使用 patch 工具精准删除
   - 确保不误删核心内容

3. **合并相似章节**
   - 将多个重复章节合并为 1 个
   - 保留最完整的描述

4. **更新版本号**
   ```yaml
   version: X.Y.Z → X.Y+1.Z
   ```

## 优化原则

### 1. 一个约束只说一次

**错误**：同一约束在 5 个章节重复
**正确**：在一个核心章节完整说明

### 2. 优先级明确

**结构**：
```markdown
## 核心强制约束（最高优先级）
### 必须执行的要求
### 绝对禁止行为
### 执行前验证清单
```

### 3. 示例要有针对性

**错误**：在约束章节重复示例
**正确**：在"标准流程"章节提供示例

### 4. 避免层级过深

**建议**：
- 最多 3 级标题（##、###、####）
- 超过 3 级说明结构不合理

## 常见问题

### 问题 1：如何判断是否重复？

**方法**：
1. 搜索关键词出现频率
2. 对比章节内容相似度
3. 检查是否表达同一意思

**标准**：
- 同一约束出现 > 2 次 → 重复
- 同一验证清单出现 > 1 次 → 重复

### 问题 2：删除后内容不完整怎么办？

**原则**：
- 保留最完整的描述
- 删除次要描述
- 如需补充，在保留章节补充

### 问题 3：如何确保不误删？

**步骤**：
1. 先读取目标章节完整内容
2. 确认无关键信息
3. 使用 patch 精准删除
4. 验证删除结果
