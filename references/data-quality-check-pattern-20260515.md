# 数据质量检测模式

**创建时间**: 2026-05-15
**场景**: 工作流执行中遇到数据质量问题

---

## 问题现象

**工作流**: 资产收集流程
**步骤**: 步骤5 - AI分析数据
**问题**: JSON文件中文内容乱码

**具体表现**:
```
标题: "��器�收_��器估价" (应为"变压器回收_变压器估价")
内容: 大量乱码字符
影响: 无法准确分析URL归属
```

---

## 检测方法

### 1. 文件编码检查
```bash
file <文件名>
# 正确: UTF-8 Unicode text
# 错误: ISO-8859, ASCII with escape sequences
```

### 2. 内容抽样检查
```bash
jq -r '.assets[] | .title' <文件名> | head -5
# 观察是否有乱码字符
```

### 3. 字符编码验证
```bash
# 检查前100字节
head -c 100 <文件名> | hexdump -C
# UTF-8 BOM: ef bb bf
# 无BOM但内容正常: 直接显示中文字符
```

---

## 处理流程

**遵循约束**: 遇到问题立即停止，不上报不诊断不修复

### 标准处理步骤

1. **检测问题**
   - 读取数据时发现异常
   - 确认问题类型（编码、格式、缺失等）

2. **停止工作流**
   - 不尝试修复
   - 不跳过步骤
   - 不使用替代方案

3. **汇报状态**
   - 问题类型
   - 影响范围
   - 当前进度
   - 建议修复方案

4. **等待指示**
   - 使用 clarify 工具询问用户
   - 根据用户选择继续或停止

---

## 常见数据质量问题

| 问题类型 | 检测方法 | 影响 |
|---------|---------|------|
| 编码错误 | file命令、内容抽样 | 中文乱码 |
| 格式错误 | jq解析、JSON验证 | 解析失败 |
| 数据缺失 | 字段检查、空值检测 | 分析不完整 |
| 数据过期 | 时间戳检查、版本验证 | 结果不准确 |

---

## 预防措施

### 下载前
- 确认远程数据源编码
- 测试下载小样本验证

### 下载时
- 指定编码参数（如 aria2c 无编码参数，需后续处理）
- 验证下载完整性

### 下载后
- 立即检查编码
- 抽样验证内容
- 记录数据来源和版本
