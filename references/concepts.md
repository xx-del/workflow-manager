# workflow-manager 核心概念

## 两个阶段

```
阶段1：设计测试 → 阶段2：创建工作流
    ↓                      ↓
  用户描述需求        测试成功后询问
  助手生成草案    →   用户确认创建
  执行测试              标准化工作流
  返回结果
```

### 阶段1：设计测试
- 用户描述需求
- 助手生成草案（WORKFLOW.md）
- 执行测试，返回结果
- 用户调试修改

### 阶段2：创建工作流
- 测试成功后用户确认
- 助手保存标准化工作流
- 自动注册到索引
- 启用状态监测和重试

## 默认配置要求

创建工作流时，以下配置**必须包含**：

### 1. 自动重试配置
```yaml
retry:
  enabled: true        # 必须为 true
  max_attempts: 3     # 默认重试3次
  interval: 60         # 重试间隔（秒）
  backoff: exponential # 指数退避
```

### 2. 通知配置
```yaml
notify:
  on_start: false
  on_complete: true   # 完成时通知
  on_fail: true       # 失败时通知
  channel: feishu
```

### 3. 统计信息
```yaml
stats:
  total_runs: 0
  success_count: 0
  fail_count: 0
  success_rate: 0%
  avg_duration: "-"
```

### 4. Guardian 配置
```yaml
guardian:
  enabled: true
  interval: 1800       # 30分钟检查一次
  stuck_threshold: 1800
  max_repair_attempts: 3
```
