# 分支工作流步骤冲突问题（2026-05-19）

## 问题现象

拼接工作流执行时，后续子工作流的步骤失败：

```
步骤 11: 备份旧文件
错误：源文件不存在（已在步骤3清理）
```

## 根本原因

**设计缺陷**：
- 分支工作流（branch workflow）由多个子工作流拼接而成
- 子工作流共享同一工作目录
- 前序子工作流的删除步骤（如步骤3）清理了目录
- 后续子工作流的依赖步骤（如步骤11）找不到源文件

**案例分析**：资产收集流程
```
电力数据子工作流：
  步骤 3: 删除旧输出（rm -f *.txt *.json）

URL分析子工作流：
  步骤 11: 备份旧文件（cp dianli.txt bk/）
  → 失败：dianli.txt 已在步骤3被删除
```

## 解决方案

### 方案 A：跳过无效步骤（临时）

子工作流步骤检测源文件是否存在：
```bash
if [ ! -f "$source_file" ]; then
    echo "跳过：源文件不存在"
    exit 0
fi
```

### 方案 B：调整工作流设计（推荐）

**设计原则**：
- 共享目录的子工作流，删除操作应放在所有子工作流之后
- 或每个子工作流使用独立工作目录
- 或在 _index.yaml 中标记步骤依赖关系，expander 自动跳过无效步骤

### 方案 C：execute.py 增强

expander.py 展开分支工作流时：
1. 检测步骤间的文件依赖
2. 标记"源文件已被删除"的步骤为 auto-skip
3. 或调整步骤顺序避免冲突

## 预防措施

1. **工作流设计时**：
   - 明确每个子工作流的工作目录
   - 避免共享目录的删除操作

2. **_index.yaml 编写时**：
   - 标注步骤的输入/输出文件
   - expander 可检测冲突

3. **执行前审计**：
   - 检查分支工作流的步骤顺序
   - 识别文件依赖冲突
