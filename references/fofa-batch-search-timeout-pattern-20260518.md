# FOFA 批量搜索超时问题与解决方案

**日期**：2026-05-18
**场景**：使用 FOFA 搜索 146 个关键词，terminal 执行超时

---

## 问题现象

**任务**：使用 FOFA 搜索 key.txt 中的 146 个关键词

**执行命令**：
```bash
cd /home/kali/zixun/konggu/xxsj && python3 main.py
```

**结果**：
- 执行时间：600 秒后超时
- 已执行批次：22 批次（约 68 个关键词）
- 未生成 JSON 报告

**错误信息**：
```
[Command timed out after 600s]
```

---

## 原因分析

### 时间计算

| 因素 | 数值 |
|------|------|
| 关键词数量 | 146 个 |
| FOFA API 查询速度 | 约 5 秒/次 |
| **预估总耗时** | **730 秒** |
| Terminal 超时限制 | 600 秒 |
| **结果** | **超时中断** |

### 执行进度

已成功执行的关键词（部分）：
- ✅ 调度中心：3 个
- ✅ EMS：28 个
- ✅ SCADA：21 个
- ✅ AGC：6 个
- ✅ AVC：7 个

- ✅ 刀闸：31 个
- ✅ 电力调度：11 个
- ✅ **储能：487 个**（最多）
- ✅ 智能电网：64 个

## 解决方案

### 方案 A：分批执行（推荐）

**优点**：
- ✅ 保留所有关键词
- ✅ 每批在超时限制内
- ✅ 可并行执行

**实施**：
```python
# 第1批：1-50个关键词
keywords_batch_1 = keywords[:50]  # 约 250 秒

# 第2批：51-100个关键词
keywords_batch_2 = keywords[50:100]  # 约 250 秒

# 第3批：101-146个关键词
keywords_batch_3 = keywords[100:]  # 约 230 秒
```

**执行流程**：
1. 创建 3 个查询文件：`queries_batch_1.txt`, `queries_batch_2.txt`, `queries_batch_3.txt`
2. 依次执行每批搜索
3. 合并结果到统一的 JSON 报告

---

### 方案 B：筛选高价值关键词

**优点**：
- ✅ 单次执行完成
- ✅ 聚焦高价值资产
- ✅ 节省 API 配额

**高价值关键词列表**（基于历史数据）：

| 关键词 | 历史命中数 | 价值等级 |
|--------|------------|----------|
| 储能 | 1765 | 超高 |
| 风电 | 1720 | 超高 |
| 光伏发电 | 523 | 超高 |
| 逆变器 | 210 | 高 |
| EMS | 297 | 高 |
| SCADA | 177 | 高 |
| 继电保护 | 24 | 中 |
| PMU | 15 | 中 |
| 智能电网 | 64 | 中 |

**筛选后关键词数**：约 20-30 个
**预估耗时**：100-150 秒（安全范围）

---

### 方案 C：增加并发数

**当前配置**：
```yaml
apis:
  fofa:
    query_size: 10000
    # 并发数：1（串行）
```

**优化配置**：
```yaml
apis:
  fofa:
    query_size: 10000
    max_concurrent: 3  # 并发数提升到 3
```

**注意**：
- ⚠️ FOFA API 有 QPS 限制（个人/专业会员 1 秒 1 次）
- ⚠️ 过高并发可能触发 429 错误
- ⚠️ 建议并发数不超过 3

## 预防措施

### 执行前预估

```python
keywords_count = 146
estimated_time = keywords_count * 5  # 秒
timeout_limit = 600  # 秒

if estimated_time > timeout_limit:
    print(f"⚠️ 预估耗时 {estimated_time} 秒，超过 {timeout_limit} 秒限制")
    print("建议：分批执行或筛选关键词")
```

### 提前告知用户

当预估耗时超过 500 秒时，应提前告知用户：

```
⚠️ 搜索预估耗时：730 秒
建议方案：
1. 分批执行（3 批次，每批约 250 秒）
2. 筛选高价值关键词（约 20 个，耗时 100 秒）

请选择方案：
```

## 相关配置

### Terminal 超时配置

**当前限制**：600 秒（10 分钟）

**调整方法**（不推荐）：
```python
terminal(command="...", timeout=900)  # 延长到 15 分钟
```

**风险**：
- ⚠️ 过长时间阻塞会话
- ⚠️ 可能影响其他任务执行
- ⚠️ 推荐使用分批执行而非延长超时

---

## 经验总结

1. **大批量操作必须预估耗时**
2. **超过 500 秒 → 分批执行**
3. **优先筛选高价值关键词**
4. **提前告知用户，避免意外中断**
5. **保留执行进度，支持断点续传**
