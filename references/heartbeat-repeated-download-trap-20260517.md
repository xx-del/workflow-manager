## 问题现象

用户问"home漏扫工作流是否执行完毕"，回答"30个报告"。

用户纠正："什么是30个呀？先分析心跳脚本"

---

## 根本原因

**心跳脚本每30分钟执行一次**，执行流程：

```
检测AWVS进度 → 完成率≥95%? → 执行步骤7下载报告
                              ↓
                    文件名含时间戳: report_2scans_20260517_183021.html
```

当AWVS保持在100%时，**每次心跳都会下载新报告**（文件名不同），导致累积：
- 00:00 → report_...000021.html
- 00:30 → report_...003020.html
- ...
- 18:30 → report_...183021.html

**这不是错误，是心跳按设计执行的结果**

---

## 心跳脚本关键逻辑

```python
# execute_step_7() 第638行
def execute_step_7():
    # 防重复检测
    step_7_status = status.get("steps", {}).get("7", {}).get("status")
    if step_7_status == "completed":
        log("⏭️ 步骤7已完成，跳过执行")
        return True
    
    # 但每次心跳重新初始化 scan_id，status文件独立
    # 所以防重复检测失效
```

每次心跳调用 `initialize_scan()` 生成新的 scan_id 和 status 文件，导致步骤状态不继承。

---

## 正确分析方法

1. **读心跳脚本** → 理解执行逻辑
2. **读心跳日志** → `tail heartbeat.log | grep "状态\|完成\|进度"`
3. **读 status.json** → 获取步骤状态
4. **区分累积产出 vs 当前轮次产出**
